# ui-minimal-fix

## 目的
UI の見た目、文言、表示有無、補助説明を、既存ロジックを壊さず最小差分で修正するための Skill である。

## この Skill を使う場面
- ボタン名やラベルが実装実態とズレている
- no-op 機能を誤解されないようにしたい
- 一時的にボタンを隠したい
- 補助説明や disabled 表示を追加したい
- 接続処理には触れず、見た目だけ整えたい

## 基本方針
- UI は実際に動いている機能名を表示する
- 実処理が無い機能名をそのまま残さない
- 文言変更、非表示、disabled 表示、補助注記を優先する
- 見た目修正のために state / join / subscribe のロジックへ広げない
- テンプレート全体の組み替えは避ける

## 音声抑制 UI の扱い
- 実態が browser 標準ノイズ抑制なら、RNNoise や DTLN という表示にしない
- `@sapphi-red/web-noise-suppressor` を導入した場合だけ、その実装に合う表示へ変更する
- 実処理が no-op の場合は、次のどちらかを優先する
  - ボタンを一時的に非表示にする
  - 文言を実態に合わせて変更する

## 守ること
- 既存の class 構造やレイアウトは必要な範囲だけ触る
- 変更対象は最小限にする
- 表示文言の変更は編集箇所だけに限定する
- 接続処理、状態管理、購読ロジックには広げない

## 禁止事項
- UI 修正のために composable / service の接続ロジックを変更する
- template 全体を書き換える
- 無関係な文言まで翻訳、変更する
- 1 行の表示調整のために大規模リファクタをする

## 推奨アプローチ
1. 実装実態とズレている UI を特定する
2. 文言変更で済むか確認する
3. 文言で足りなければ補助説明や disabled 表示を加える
4. それでも誤認が強い場合だけ一時非表示を選ぶ
5. 接続処理に触れず完了できる形を優先する

## 検証
最低でも次を確認する。
- 変更前後で主要導線が壊れていない
- 表示名と実装実態が一致している
- ボタン表示有無が意図どおり
- 既存の join / leave / share / blur 操作が影響を受けていない

## 完了報告
完了時は次を簡潔に書く。
- 変更した UI
- 文言変更か、非表示か、補助表示か
- 接続ロジックへ触っていないこと
- 未検証項目
