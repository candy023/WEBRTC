name: ui-minimal-fix
description: Vue UI の表示崩れ、文言修正、サイズ調整、クラス調整、軽い条件表示の修正を最小差分で行うときに使う。状態管理や service 層まで広げない。
---

# 目的
見た目や表示文言の修正を、接続処理や状態管理へ波及させず最小差分で行う。

# 使う場面
- テキスト修正
- ボタンやラベルの表示修正
- box が大きすぎる、小さすぎる
- 余白、幅、高さ、aspect ratio の局所調整
- 条件表示の軽い修正
- 日本語 UI の局所統一
- 共有中 / 非共有中の表示切り替え修正
- PC とモバイルで見え方の優先順位を揃えたいとき

# 基本方針
- template と class の必要箇所だけ直す
- script 側へ広げるのは本当に必要な場合だけ
- service や composable を巻き込まない
- レイアウト目的で template 全体を書き換えない
- 編集箇所の visible text だけ変える
- 日本語 UI が要求されたら編集箇所だけ日本語へ統一する
- UI 修正のために接続順序や publish / subscribe ロジックを触らない

# webrtc3 で見ること
- 共有中でないときのプレースホルダが大きすぎないか
- main share view と participant tile のサイズ責務が混ざっていないか
- `object-cover` と `object-contain` の使い分けが壊れていないか
- 横スクロール filmstrip の最小幅が大きすぎないか
- 画面共有中だけ出す UI が非共有時にも残っていないか
- ローカルプレビューとリモートタイルの見た目責務が混ざっていないか
- text 修正のために無関係な文言まで変えていないか

# レスポンシブ UI ルール
- モバイルでは、主要映像が横スクロールしないと見えない構成にしない
- モバイルでは、共有画面が存在する場合は共有画面を最優先で上部に固定する
- モバイルでは、参加者カメラを横フィルムストリップより縦積みまたはグリッド表示で優先する
- PC では、大きい主表示 + 補助サムネイル / フィルムストリップ構成を維持してよい
- 共有画面と参加者カメラは同じ一覧に戻さず、現在の責務分離を維持する
- 主要情報は無スクロールまたは縦スクロールだけで到達できる構成を優先する
- 端末差によって「見えていないだけ」を接続不具合と誤認しやすい UI を避ける

# よくある修正方針
## プレースホルダが大きすぎる
- 親コンテナ全体ではなく対象 box の幅・高さだけ調整する
- 共有エリアのレイアウトと placeholder のレイアウトを分けて考える
- 画面共有中と非共有中で同じサイズ指定を使い回しすぎていないか確認する

## 共有画面が大きすぎる / 小さすぎる
- share 専用コンテナの class を見直す
- participant tile 側の class 調整で代用しない
- `aspect-video` や `object-contain` の責務を崩さない

## モバイルで横スワイプしないと見えない
- モバイルでは横フィルムストリップを前提にしない
- 主表示は上、補助表示は下の縦方向の情報設計を優先する
- 参加者カメラは縦積みまたは 2 列グリッドを優先する
- `md` 以上だけ既存の横フィルムストリップ構成を維持する

## 文言だけ変えたい
- visible text の必要箇所だけ変える
- script や state 名には触れない
- 他画面の用語まで一括変更しない

# 禁止事項
- 見た目修正のために state 管理まで広げない
- 無関係な文言までまとめて変えない
- class 整理を理由に広く reformat しない
- layout のために component 構造を大きく変えない
- UI 修正に便乗して service/composable の責務を変えない
- 接続不具合と表示優先順位の問題を混同しない

# 完了条件
- 指摘された見た目だけが直っている
- 既存の状態制御は変わっていない
- 差分が template / class / 必要最小限の条件式に収まっている
- 編集箇所の UI 言語が依頼どおりになっている
- モバイルで主要映像が初見で見える
